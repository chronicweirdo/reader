Title of the chapter

As all things go, this will be a test chapter. Written in the peaceful dusk of a late autumn afternoon, this text will be used to look into three main functionalities required for an ebook parsing algorithm. Three, the magic number, that is not mere two, and nowhere as far as four main functionalities, but a modest three.

Parsing to tree

Test if we can parse document to a tree of nodes. Leaves hold content. Leaves can be text nodes, images, empty tags and maybe tables.

Images, empty tags for spacing and maybe tables have a size of 1, meaning they can't be split in multiple pages.

Compute sizes to be used in the page computation algorithm and when computing book reading progress.

Splitting the tree into chunks

Functionality for copying parts of the tree, while keeping the structure of the tree.

Used when splitting the document into pages.

Computing pages

Functionality for finding spaces. Getting the previous and next position of a space will allow us to compute pages without splitting mid-word.
